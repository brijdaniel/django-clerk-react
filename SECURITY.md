# Security Policy

## Reporting a vulnerability

If you believe you have found a security vulnerability in an application
built from this template, please report it privately to the address below.
Please do not open a public issue or disclose the vulnerability publicly
until the maintainers have had a chance to investigate and release a fix.

> **Replace this contact before going to production:**
> **security@example.com**

When reporting, include where possible:

- A description of the vulnerability and its impact
- Steps to reproduce (a proof of concept helps)
- The affected endpoint, page, or component
- Any suggested remediation

## What to expect

- We will acknowledge your report within 3 business days.
- We will keep you informed of progress while we investigate and remediate.
- We will credit you in the fix notes if you would like (and the report is
  valid), once a fix has shipped.

## Scope notes

- List your production hostnames here (e.g. `app.example.com`,
  `api.example.com`) so researchers know what is in scope.
- Please do not run automated scanners against production, attempt denial of
  service, or access data that is not your own. Use a test account wherever
  possible.
